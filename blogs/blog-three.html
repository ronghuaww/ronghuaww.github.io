What did you think of assertions, unit tests, coverage, is_prime() and argument passing?

Assertions were a coding statement that I previously knew about, but had difficulty understanding the purpose and usage of assertions in the code. Argument passing was a difficult concept to wrap my head around during the first lecture. After seeing so many examples in the second example, I gain a better understanding of addresses and pointers.

What made you happy this week?

I finally understood the difference between addresses and pointers. Admittedly, I used to blindly add pointers and ampersands until it worked.

What's your pick-of-the-week or tip-of-the-week?

Consistently use “TODO” at the beginning of a comment when indicating a part of code that needs to be revisited. That way, all the tasks needed to be done can be found by searching for “TODO” in the search bar. Many IDE also highlights these comments that start with “TODO”, making them easy to find in a given file.
